## 0.1 这份课件是写给谁的

这份课件是写给 **Agent 开发小白** 的。

什么叫「Agent 开发小白」？我们假设你：

- ✅ 会写基本的 Python（懂函数、类、字典、列表）
- ✅ 调用过 LLM 的 API（比如用 `openai` 库发过一条消息，或者至少知道「发请求 → 收回复」这件事）
- ✅ 能看懂 YAML 和 JSON 配置文件
- ✅ 会在命令行里敲命令（`cd`、`git clone`、`pip install`）
- ❌ 没有系统学过 Agent 框架（不知道 LangChain、LangGraph 是什么也没关系）
- ❌ 没有从头搭建过一个能「自己动手干活」的 AI 系统

如果你的情况符合上面，那么恭喜你，这正是为你量身定制的。

如果你已经是个 Agent 老手，可以直接跳到第 03 章看架构总览，再按需挑选章节。

## 0.2 我们要学什么：deer-flow 是什么

**deer-flow**（DeerFlow）是字节跳动开源的一个 **AI 超级 Agent 运行框架（Super Agent Harness）**。

别被「超级」两个字吓到，我们先用一个类比把它拆开：

| 类比 | 现实世界 | deer-flow 世界 |
|------|---------|---------------|
| 一个聪明的员工 | 大脑（LLM） | 大模型（GPT、DeepSeek、豆包…） |
| 员工的手和脚 | 执行能力 | **工具系统**（搜索、读写文件、跑代码） |
| 员工的工位 | 工作环境 | **沙箱系统**（一个隔离的「小电脑」） |
| 员工的同事 | 协作 | **子代理系统**（把任务外包给副手） |
| 员工的笔记本 | 记忆 | **记忆系统**（跨对话记住你的喜好） |
| 员工的技能培训 | 专业能力 | **技能系统（Skills）**（一份 Markdown 定义一个能力） |

普通聊天机器人只有「大脑」，你问它答。而 deer-flow 给大脑配上了一整套「身体」，让它能**真正干活**：不只是告诉你怎么做 PPT，而是**直接帮你做出一份 PPT 文件**。

> 💡 **一句话理解 deer-flow**：它不是一个聊天机器人，而是一个「会自己动手的 AI 员工运行平台」。

学完本课件，你会清楚地知道这套「身体」是怎么搭起来的，并且有能力自己动手搭一个。

---

## 0.3 为什么选 deer-flow 作为学习对象

市面上 Agent 框架很多（LangChain、AutoGPT、CrewAI…），为什么我们选 deer-flow？

1. **工程化程度高**：它是字节跳动团队打磨的生产级项目（2026 年 2 月曾登顶 GitHub Trending 第一），不是玩具 demo。代码规范、分层清晰，非常适合学习「真正的 Agent 系统怎么写」。

2. **建立在 LangGraph 之上**：LangGraph 是当前最主流的 Agent 编排框架之一。学懂 deer-flow，等于顺带掌握了 LangGraph 的实战用法。

3. **功能完整**：工具、沙箱、子代理、记忆、技能、多渠道接入……一个 Agent 系统该有的它全有，是学习「完整 Agent 架构」的最佳样本。

4. **可扩展**：它被设计成「可以拆开改造」的，每一层都能替换。这正好适合我们「学懂一个模块 → 自己动手扩展一个」的学习方式。

## 0.5 每章的统一结构

为了让你的学习体验一致，后续每一章都遵循下面的结构：

```
1. 学习目标        ← 这章学完你能做什么（3-5 条明确的收获）
2. 概念铺垫        ← 用类比和图解释，先不上代码
3. 核心原理        ← 结合 deer-flow 的架构讲清楚原理
4. 代码精读        ← 贴真实代码 + 逐行注释 + 伪代码图解
5. 动手实验        ← 可选的实操步骤（部分章节有）
6. 小结与检查点    ← 关键概念回顾，自测是否掌握
7. 下一章预告      ← 承上启下
```

其中 **「代码精读」是核心**。我们承诺不吝啬文字和代码——每个关键实现都会贴出 deer-flow 的真实代码，逐行讲清楚「它在干什么、为什么这么写」。这是这份课件区别于其他「走马观花式」教程的最大特点。
