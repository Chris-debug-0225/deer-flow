简介

上下文工程是提升 Agent 智能的关键。通过精心设计 Prompt、管理上下文窗口、优化信息流，可以显著提高 Agent 的理解能力和决策质量。本章讲解上下文工程的核心概念、Prompt 设计、上下文管理和性能优化等内容。

学习目标：
  • 理解上下文对 Agent 性能的影响
  • 掌握 Prompt 工程的最佳实践
  • 学会管理和优化上下文
  • 提升 Agent 的智能水平

上下文的重要性

上下文是 Agent 理解和决策的基础。好的上下文能让 Agent 更准确地理解问题，做出更好的决策。

上下文的作用

  • 背景信息：提供问题的背景和相关信息
  • 指导方向：引导 Agent 朝着正确的方向思考
  • 约束条件：限制 Agent 的行为范围
  • 示例学习：通过示例帮助 Agent 学习模式
  • 角色定位：定义 Agent 的角色和职责

上下文窗口

模型 上下文窗口 优势 劣势
GPT-3.5 4K tokens 成本低 上下文有限
GPT-4 8K/32K tokens 上下文较大 成本高
Claude 3 200K tokens 超大上下文 成本更高
关键点： 上下文窗口大小直接影响 Agent 能处理的信息量。选择合适的模型很重要。

Prompt工程

Prompt 是与 Agent 沟通的语言。好的 Prompt 能让 Agent 更好地理解需求。

Prompt 设计原则

# 原则 1: 清晰明确 "请分析这个数据" # 不好：模糊 "请分析这个销售数据，找出增长最快的产品类别" # 好：清晰 # 原则 2: 提供上下文 "你是一个数据分析师。请分析2024年的销售数据..." # 原则 3: 指定输出格式 "请返回 JSON 格式的结果，包含 'product', 'growth_rate', 'rank' 字段" # 原则 4: 使用示例 "示例输入: {...}, 示例输出: {...}"

Prompt 模板

class PromptTemplate: """Prompt 模板""" def __init__(self): self.system_prompt = """你是一个专业的数据分析师。 你的职责是： 1. 分析数据 2. 识别趋势 3. 提供建议 """ def build_prompt(self, task: str, data: str) -> str: """构建完整的 Prompt""" return f"""{self.system_prompt} 任务：{task} 数据： {data} 请提供详细的分析结果。 """
示例： 好的 Prompt 设计
"""你是一个专业的代码审查专家。 任务：审查以下 Python 代码，找出潜在的问题。 代码： ```python def calculate_total(items): total = 0 for item in items: total = total + item['price'] * item['quantity'] return total ``` 请提供： 1. 代码质量评分（1-10） 2. 发现的问题列表 3. 改进建议 4. 改进后的代码示例 输出格式：JSON """

上下文管理

管理上下文意味着有效地组织和利用有限的上下文窗口。

上下文管理策略

收集信息
→
优先级排序
→
选择关键信息
↓
构建 Prompt
→
发送请求
→
获取结果

上下文管理的实现

class ContextManager: """上下文管理器""" def __init__(self, max_tokens: 4000): self.max_tokens = max_tokens self.context = [] def add_context(self, text: str, priority: 1): """添加上下文""" tokens = len(text.split()) self.context.append({ "text\": text, "tokens\": tokens, "priority\": priority }) def get_context(self) -> str: """获取优化后的上下文""" # 按优先级排序 sorted_context = sorted( self.context, key=lambda x: x["priority\"], reverse=True ) # 选择不超过 max_tokens 的内容 total_tokens = 0 selected = [] for item in sorted_context: if total_tokens + item["tokens\"] <= self.max_tokens: selected.append(item["text\"]) total_tokens += item["tokens\"] return "\n".join(selected)

高级技巧

除了基础的上下文管理，还有一些高级技巧可以进一步提升 Agent 的性能。

思维链（Chain of Thought）

要求 Agent 逐步思考和推理，而不是直接给出答案：

"""请按以下步骤解决这个问题： 1. 首先，理解问题的核心 2. 其次，列出可能的解决方案 3. 然后，分析每个方案的优缺点 4. 最后，给出最优方案和理由 问题：如何优化数据库查询性能？ """

角色扮演（Role Playing）

定义 Agent 的角色，让它在特定的角色背景下工作：

class RoleBasedPrompt: """基于角色的 Prompt""" def __init__(self, role: str): self.role = role def build(self, task: str) -> str: """构建角色 Prompt""" return f"""你是一个{self.role}。 根据你的专业知识和经验，完成以下任务： {task} 请用{self.role}的角度和方式来回答。 """

检索增强生成（RAG）

结合外部知识库，提供更准确的上下文：

class RAGSystem: """检索增强生成系统""" def __init__(self, knowledge_base): self.kb = knowledge_base def retrieve_context(self, query: str, top_k: 5) -> List[str]: """从知识库检索相关上下文""" # 使用向量相似度搜索 results = self.kb.search(query, top_k=top_k) return [r['content'] for r in results] def augment_prompt(self, query: str) -> str: """用检索到的上下文增强 Prompt""" context = self.retrieve_context(query) return f"""根据以下背景信息回答问题： 背景信息： {chr(10).join(context)} 问题：{query} """

动态上下文调整

根据 Agent 的反馈动态调整上下文：

class DynamicContextAdjuster: """动态上下文调整器""" def adjust(self, response: str, confidence: 0.5) -> str: """根据置信度调整上下文""" if confidence < 0.5: # 置信度低，添加更多上下文 return "增加详细信息和示例" elif confidence > 0.9: # 置信度高，简化上下文 return "简化上下文，提高效率" else: # 置信度中等，保持当前上下文 return "保持当前上下文"

性能优化

优化上下文可以提高 Agent 的性能和降低成本。

优化技巧

优化方法 说明 效果
摘要 将长文本摘要为关键信息 减少 50-70% tokens
分块 将大任务分成小任务 提高准确性
缓存 缓存常用的上下文 减少 API 调用
检索增强 只检索相关的信息 提高相关性
警告： 过度优化可能导致丢失重要信息。需要在效率和准确性之间找到平衡。

最佳实践

1. 系统提示词

使用系统提示词定义 Agent 的角色和行为。这比在每个请求中重复说明更高效。

2. 示例学习

提供具体的示例帮助 Agent 理解期望的输出格式和风格。

3. 逐步推理

要求 Agent 逐步思考和解释，而不是直接给出答案。这提高了准确性。

4. 反馈循环

根据 Agent 的输出进行调整，不断优化 Prompt 和上下文。

5. 版本管理

保存不同版本的 Prompt，跟踪哪个版本效果最好。
