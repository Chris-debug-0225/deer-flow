简介

自定义 Skill 是扩展 Agent 功能的核心方式。通过编写自定义 Skill，可以让 Agent 执行任何你想要的操作。本章讲解如何从零开始编写、测试和部署自定义 Skill。

学习目标：
  • 理解 Skill 的结构和生命周期
  • 掌握编写自定义 Skill 的完整流程
  • 学会测试和调试 Skill
  • 了解部署和维护 Skill

Skill 结构

每个 Skill 都有标准的结构和接口。理解这个结构是编写 Skill 的基础。

Skill 的基本组成

from abc import ABC, abstractmethod from typing import Any, Dict class Skill(ABC): """Skill 基类""" def __init__(self, name: str, description: str): self.name = name self.description = description self.version = "1.0.0" @abstractmethod def execute(self, **kwargs) -> Any: """执行 Skill""" pass @abstractmethod def validate_input(self, **kwargs) -> bool: """验证输入参数""" pass def get_metadata(self) -> Dict: """获取 Skill 元数据""" return { "name": self.name, "description": self.description, "version": self.version }

实现步骤

编写自定义 Skill 的完整步骤。

第 1 步：定义 Skill 类

class DataAnalysisSkill(Skill): """数据分析 Skill""" def __init__(self): super().__init__( name="data_analysis", description="分析数据并生成报告" ) def validate_input(self, **kwargs) -> bool: """验证输入""" required_fields = ["data", "analysis_type"] return all(field in kwargs for field in required_fields) def execute(self, **kwargs): """执行分析""" if not self.validate_input(**kwargs): raise ValueError("Invalid input") data = kwargs["data"] analysis_type = kwargs["analysis_type"] # 执行分析逻辑 result = self.analyze(data, analysis_type) return result def analyze(self, data, analysis_type): """分析数据""" if analysis_type == "summary": return self.summarize(data) elif analysis_type == "trend": return self.analyze_trend(data) else: raise ValueError("Unknown analysis type") def summarize(self, data): """生成摘要""" return {"type": "summary", "data": data} def analyze_trend(self, data): """分析趋势""" return {"type": "trend", "data": data}
示例： 完整的 Skill 实现
# 使用 Skill skill = DataAnalysisSkill() # 执行 Skill result = skill.execute( data=[1, 2, 3, 4, 5], analysis_type="summary" ) print(result) # 输出: {'type': 'summary', 'data': [1, 2, 3, 4, 5]}

高级特性

除了基本功能，Skill 还支持一些高级特性来增强功能和性能。

异步执行

对于耗时操作，可以使用异步执行提高效率：

import asyncio class AsyncDataAnalysisSkill(Skill): async def execute_async(self, **kwargs): """异步执行""" return await self.analyze_async(kwargs) async def analyze_async(self, kwargs): """异步分析""" await asyncio.sleep(1) # 模拟耗时操作 return {"status": "completed"}

缓存和记忆化

为 Skill 添加缓存以提高性能：

from functools import lru_cache class CachedSkill(Skill): @lru_cache(maxsize=128) def expensive_operation(self, key: str): """昂贵的操作，结果会被缓存""" # 执行昂贵的计算 return f"result_for_{key}"

错误恢复

实现错误恢复机制确保 Skill 的健壮性：

class RobustSkill(Skill): def execute_with_retry(self, **kwargs): """带重试的执行""" max_retries = 3 for attempt in range(max_retries): try: return self.execute(**kwargs) except Exception as e: if attempt == max_retries - 1: raise print(f"Retry {attempt + 1}")

测试

充分的测试是确保 Skill 质量的关键。

单元测试

import unittest class TestDataAnalysisSkill(unittest.TestCase): def setUp(self): self.skill = DataAnalysisSkill() def test_valid_input(self): """测试有效输入""" result = self.skill.execute( data=[1, 2, 3], analysis_type="summary" ) self.assertIsNotNone(result) def test_invalid_input(self): """测试无效输入""" with self.assertRaises(ValueError): self.skill.execute(data=[1, 2]) def test_metadata(self): """测试元数据""" metadata = self.skill.get_metadata() self.assertEqual(metadata["name"], "data_analysis")

部署

部署 Skill 使其可以被 Agent 使用。

部署步骤

  1. 打包： 将 Skill 代码打包为模块
  2. 注册： 在 Agent 中注册 Skill
  3. 验证： 验证 Skill 可以正常使用
  4. 监控： 监控 Skill 的执行情况

注册 Skill

class Agent: def __init__(self): self.skills = {} def register_skill(self, skill: Skill): """注册 Skill""" self.skills[skill.name] = skill def execute_skill(self, skill_name: str, **kwargs): """执行 Skill""" if skill_name not in self.skills: raise ValueError(f"Skill {skill_name} not found") skill = self.skills[skill_name] return skill.execute(**kwargs) # 使用示例 agent = Agent() agent.register_skill(DataAnalysisSkill()) result = agent.execute_skill("data_analysis", data=[1, 2, 3], analysis_type="summary")

最佳实践

1. 清晰的接口

Skill 的接口应该清晰、简单。参数和返回值应该有明确的类型和说明。

2. 错误处理

实现完善的错误处理。验证输入，捕获异常，提供有意义的错误信息。

3. 文档

为 Skill 编写清晰的文档。说明参数、返回值、异常和使用示例。

4. 版本管理

维护 Skill 的版本。当 API 改变时更新版本号。

5. 性能考虑

优化 Skill 的性能。考虑缓存、异步执行等优化方法。

关键点： 好的 Skill 应该是可复用、可测试、易维护的。
