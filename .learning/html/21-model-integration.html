简介

不同的 LLM 模型各有优缺点。支持多种模型能让 Agent 根据需求灵活选择最合适的模型。本章讲解如何集成多个 LLM、设计模型抽象层、实现模型切换和成本优化等内容。

学习目标：
  • 理解不同 LLM 的特点和差异
  • 掌握模型抽象层的设计
  • 学会实现模型切换机制
  • 了解成本优化策略

模型抽象层

设计一个抽象层可以隐藏不同模型的差异，提供统一的接口。

抽象层设计

from abc import ABC, abstractmethod class LLMProvider(ABC): """LLM 提供商抽象基类""" @abstractmethod def invoke(self, prompt: str, **kwargs) -> str: """调用模型""" pass @abstractmethod def get_cost(self, tokens: 1000) -> 0.0: """计算成本""" pass # 具体实现 class OpenAIProvider(LLMProvider): def __init__(self, api_key: str, model: str = "gpt-3.5-turbo"): self.api_key = api_key self.model = model def invoke(self, prompt: str, **kwargs) -> str: # 调用 OpenAI API pass def get_cost(self, tokens: 1000) -> 0.0: # 计算 OpenAI 成本 pass class AnthropicProvider(LLMProvider): def __init__(self, api_key: str, model: str = "claude-3-opus"): self.api_key = api_key self.model = model def invoke(self, prompt: str, **kwargs) -> str: # 调用 Anthropic API pass def get_cost(self, tokens: 1000) -> 0.0: # 计算 Anthropic 成本 pass

模型选择器实现

class ModelSelector: """模型选择器""" def __init__(self): self.providers = { "gpt-4": OpenAIProvider("key", "gpt-4"), "gpt-3.5": OpenAIProvider("key", "gpt-3.5-turbo"), "claude": AnthropicProvider("key") } def select_model(self, task_complexity: 5, budget: 10) -> LLMProvider: """根据任务复杂度和预算选择模型""" if task_complexity > 8: return self.providers["gpt-4"] elif budget < 5: return self.providers["gpt-3.5"] else: return self.providers["claude"]
示例： 模型切换的实际应用
selector = ModelSelector() # 简单任务：使用便宜的模型 simple_task = "翻译这句话" model = selector.select_model(task_complexity=2, budget=1) result = model.invoke(simple_task) # 复杂任务：使用强大的模型 complex_task = "分析这个复杂的代码并提出优化建议" model = selector.select_model(task_complexity=9, budget=50) result = model.invoke(complex_task)

故障转移策略

当某个模型失败或超时时，需要有备用方案来保证系统的可靠性。

故障转移实现

class FallbackModelManager: """故障转移模型管理器""" def __init__(self): self.primary_model = OpenAIProvider("key", "gpt-4") self.fallback_models = [ AnthropicProvider("key"), OpenAIProvider("key", "gpt-3.5-turbo") ] def invoke_with_fallback(self, prompt: str, timeout: 30) -> str: """尝试使用主模型，失败时使用备用模型""" models = [self.primary_model] + self.fallback_models for model in models: try: result = model.invoke(prompt, timeout=timeout) return result except Exception as e: # 记录错误，尝试下一个模型 print(f"Model failed: {e}, trying fallback") continue raise Exception("All models failed")

成本优化

使用多个模型可以优化成本。关键是在成本和质量之间找到平衡。

成本优化策略

  • 分级处理：简单任务用便宜模型，复杂任务用强大模型
  • 缓存结果：缓存常见问题的答案，避免重复调用
  • 批量处理：合并多个请求，减少 API 调用次数
  • 降级策略：当高级模型超时或失败时，降级到低级模型
  • 成本监控：实时监控成本，设置预算告警

成本计算器

class CostCalculator: """成本计算器""" def __init__(self): self.rates = { "gpt-4": {"input": 0.03, "output": 0.06}, "gpt-3.5": {"input": 0.0005, "output": 0.0015}, "claude": {"input": 0.008, "output": 0.024} } def calculate_cost(self, model: str, input_tokens: 1000, output_tokens: 500) -> 0.0: """计算调用成本""" rate = self.rates[model] input_cost = input_tokens * rate["input"] / 1000 output_cost = output_tokens * rate["output"] / 1000 return input_cost + output_cost
关键点： 成本优化不是简单地选择最便宜的模型，而是在成本、质量和速度之间找到最优平衡。

最佳实践

1. 统一接口

为所有模型提供统一的接口，使得切换模型不需要改变上层代码。

2. 错误处理

实现降级策略，当一个模型失败时自动切换到备用模型。

3. 性能监控

监控每个模型的性能指标，包括延迟、成本和准确性。

4. 配置管理

使用配置文件管理模型选择策略，便于调整而不需要修改代码。

5. 成本控制

设置成本预算和告警，防止意外的高成本。

总结

支持多种 LLM 模型能让 Agent 更灵活地适应不同场景。通过设计良好的抽象层、实现智能的模型选择、故障转移和成本优化，可以构建高效、经济、可靠的 Agent 系统。本章深入讲解了模型集成的各个方面。

核心要点总结：
  • 模型生态： 了解不同模型的优缺点和成本
  • 抽象层设计： 为所有模型提供统一接口
  • 模型切换： 根据任务复杂度和预算动态选择模型
  • 故障转移： 当模型失败时自动切换到备用模型
  • 成本优化： 分级处理、缓存、批量处理、降级策略
  • 最佳实践： 统一接口、错误处理、性能监控、配置管理、成本控制
模型集成建议： 多模型集成是构建鲁棒 Agent 系统的关键。需要在性能、成本和可靠性之间找到平衡。同时要定期评估模型性能，根据实际情况调整策略。
