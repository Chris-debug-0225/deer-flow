简介

Skills 是 Agent 的能力库。一个 Agent 本身什么都做不了，只有通过 Skills 才能调用外部工具、查询数据库、发送邮件等。本章讲解 Skills 的定义、注册、调用机制，以及如何扩展 Agent 的能力。

学习目标：
  • 理解 Skill 的概念和作用
  • 学会定义和注册 Skill
  • 掌握 Skill 的调用机制
  • 了解能力扩展的最佳实践

Skill 的概念

Skill 是 Agent 可以调用的一个能力或工具。它包装了一个具体的功能（如查询天气、发送邮件、查询数据库等），并提供了统一的接口供 Agent 调用。

Skill 的核心要素

名称 Skill 的唯一标识 get_weather, send_email
描述 Skill 的功能说明，供 LLM 理解 "查询指定城市的天气信息"

内置 Skill 是 DeerFlow 提供的常用能力，如 HTTP 请求、数据库查询等。自定义 Skill 是开发者根据业务需求编写的特定能力。

Skill 的定义与注册

定义一个 Skill 需要指定其名称、描述、参数和实现函数。DeerFlow 提供了装饰器简化这个过程。

使用装饰器定义 Skill

from deerflow import Skill, skill @skill( name="get_weather", description="查询指定城市的天气信息" ) def get_weather(city: str, date: str = "today") -> str: """查询天气的具体实现 Args: city: 城市名称 date: 日期，默认为今天 Returns: 天气信息字符串 """ # 这里是实际的实现逻辑 return f"{city} 今天天气晴朗，温度 25°C"

手动创建 Skill 对象

如果不想使用装饰器，也可以手动创建 Skill 对象：

from deerflow import Skill def send_email(to: str, subject: str, body: str) -> bool: """发送邮件的实现""" # 邮件发送逻辑 return True # 创建 Skill 对象 email_skill = Skill( name="send_email", description="发送电子邮件", func=send_email, parameters={ "to": {"type": "string", "description": "收件人邮箱"}, "subject": {"type": "string", "description": "邮件主题"}, "body": {"type": "string", "description": "邮件内容"} } )

向 Agent 注册 Skill

定义好 Skill 后，需要将其注册到 Agent，这样 Agent 才能调用它：

from deerflow import Agent # 创建 Agent agent = Agent( name="weather_assistant", llm=llm, system_prompt="你是一个天气助手" ) # 注册 Skill agent.add_skill(get_weather) agent.add_skill(send_email) # 或者一次性注册多个 Skill agent.add_skills([get_weather, send_email])
关键点： Skill 的描述非常重要，因为 LLM 会根据描述来判断是否需要调用这个 Skill。好的描述应该清晰、简洁、准确。

Skill 的调用机制

当 Agent 执行时，LLM 会根据用户输入和 Skill 的描述，决定是否调用某个 Skill。执行引擎负责解析 LLM 的决策，调用相应的 Skill，并将结果返回给 LLM。

Skill 调用的实现

class SkillExecutor: def __init__(self, skills: Dict[str, Skill]): self.skills = skills def execute(self, skill_name: str, **kwargs) -> str: """执行指定的 Skill""" if skill_name not in self.skills: raise SkillNotFoundError( f"Skill '{skill_name}' not found" ) skill = self.skills[skill_name] # 验证参数 self.validate_parameters(skill, kwargs) # 执行 Skill try: result = skill.func(**kwargs) return str(result) except Exception as e: return f"Error: {str(e)}" def validate_parameters(self, skill: Skill, params: Dict): """验证参数是否符合 Skill 的要求""" required_params = skill.required_parameters for param in required_params: if param not in params: raise ValueError( f"Missing required parameter: {param}" )
示例： 完整的 Skill 调用过程
# 1. 定义 Skill @skill(name="calculate", description="计算两个数的和") def calculate(a: int, b: int) -> int: return a + b # 2. 创建 Agent 并注册 Skill agent = Agent(llm=llm) agent.add_skill(calculate) # 3. 用户输入 user_input = "5 加 3 等于多少？" # 4. Agent 执行 result = agent.run(user_input) # LLM 会识别需要调用 calculate Skill # 执行引擎调用 calculate(5, 3) # 返回结果 "8"

Skill 的类型与分类

根据不同的维度，Skill 可以分为多种类型。了解这些类型有助于更好地设计和使用 Skill。

按功能分类

类型 说明 示例
数据查询 从外部系统查询数据 查询天气、查询数据库、搜索网页
数据修改 修改或创建数据 创建订单、更新用户信息、删除记录
外部服务 调用第三方服务 发送邮件、发送短信、支付处理
计算处理 本地计算和数据处理 数据分析、文本处理、格式转换
系统操作 系统级别的操作 文件操作、进程管理、系统监控

按执行方式分类

Skill 可以是同步的也可以是异步的：

# 同步 Skill @skill(name="sync_query", description="同步查询") def sync_query(query: str) -> str: # 直接返回结果 return "query result" # 异步 Skill @skill(name="async_query", description="异步查询") async def async_query(query: str) -> str: # 使用 await 等待异步操作 result = await fetch_data(query) return result

按权限级别分类

不同的 Skill 可能需要不同的权限级别：

@skill( name="read_file", description="读取文件", permission_level="read" ) def read_file(path: str) -> str: with open(path) as f: return f.read() @skill( name="delete_file", description="删除文件", permission_level="write" ) def delete_file(path: str) -> bool: import os os.remove(path) return True

生命周期钩子

DeerFlow 提供了生命周期钩子，允许在不同阶段执行自定义逻辑：

class DatabaseSkill: """数据库操作 Skill，演示生命周期钩子""" def __init__(self): self.connection = None def on_init(self): """初始化时调用，建立数据库连接""" self.connection = connect_to_database() def on_execute(self, query: str) -> str: """执行查询""" result = self.connection.execute(query) return str(result) def on_cleanup(self): """清理时调用，关闭数据库连接""" if self.connection: self.connection.close()

Skill 的测试与调试

在将 Skill 部署到生产环境之前，应该进行充分的测试和调试。DeerFlow 提供了测试工具和调试功能。

单元测试

为 Skill 编写单元测试，确保其功能正确：

import unittest from deerflow import Skill class TestWeatherSkill(unittest.TestCase): def setUp(self): self.skill = get_weather def test_valid_city(self): """测试有效的城市""" result = self.skill.execute(city="北京") self.assertIsNotNone(result) self.assertIn("北京", result) def test_invalid_city(self): """测试无效的城市""" with self.assertRaises(ValueError): self.skill.execute(city="不存在的城市")

集成测试

测试 Skill 与 Agent 的集成：

def test_skill_with_agent(): """测试 Skill 与 Agent 的集成""" # 创建 Agent agent = Agent(llm=llm) agent.add_skill(get_weather) # 测试 Agent 是否能正确调用 Skill result = agent.run("北京今天天气怎么样？") # 验证结果 assert "北京" in result assert "天气" in result

调试技巧

使用以下技巧调试 Skill 的问题：

启用日志： 在 Skill 执行前后添加日志，记录输入和输出。
参数验证： 在 Skill 开始执行前验证所有参数，提供清晰的错误信息。
模拟测试： 使用 Mock 对象模拟外部依赖，进行隔离测试。
性能分析： 使用性能分析工具识别 Skill 执行的瓶颈。

Skill 设计的最佳实践

1. 清晰的命名和描述

Skill 的名称应该简洁明了，描述应该准确表达功能。好的描述能帮助 LLM 更准确地判断何时调用。

2. 参数设计

参数应该尽可能简单，避免复杂的嵌套结构。每个参数都应该有清晰的类型和说明。

3. 错误处理

Skill 应该妥善处理错误，返回有意义的错误信息，而不是让异常传播。

4. 返回值格式

Skill 的返回值应该是字符串或可以转换为字符串的对象。如果返回复杂的数据结构，应该转换为 JSON 字符串。

5. 性能考虑

Skill 的执行时间应该在合理范围内。对于耗时操作，考虑添加超时机制。

警告： 不要在 Skill 中执行危险操作（如删除数据库、修改系统文件等）而不进行充分的验证和确认。应该实现权限检查和审计日志。

总结

Skills 是 Agent 的能力库，通过 Skills，Agent 可以调用各种外部工具和服务。本章深入讲解了 Skill 的定义、注册、调用、分类、生命周期、测试和最佳实践。

核心要点总结：
  • Skill 的概念： 包装了具体功能的能力单元，有名称、描述、参数和实现函数
  • 定义和注册： 使用装饰器或手动创建 Skill 对象，然后注册到 Agent
  • 调用机制： LLM 根据描述决定调用，执行引擎负责参数验证和执行
  • Skill 分类： 按功能、执行方式、权限级别等多个维度分类
  • 生命周期管理： 理解定义、注册、初始化、执行、清理各阶段，使用生命周期钩子
  • 测试与调试： 编写单元测试和集成测试，使用日志和模拟工具进行调试
  • 最佳实践： 清晰的命名、简单的参数、完善的错误处理、合理的性能、充分的安全验证
实践建议： 在设计 Skill 时，始终考虑 LLM 的理解能力。好的 Skill 描述应该让 LLM 能够准确判断何时调用。同时，要实现完善的错误处理和安全验证，确保 Skill 的可靠性和安全性。
