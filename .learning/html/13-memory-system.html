简介

长期记忆是 Agent 的知识库。没有记忆，Agent 每次都要从零开始；有了记忆，Agent 可以学习、积累经验、改进决策。本章讲解记忆的存储方式、检索机制、更新策略，以及记忆对 Agent 性能的影响。

学习目标：
  • 理解记忆系统的重要性
  • 掌握记忆的存储和检索机制
  • 学会实现记忆的更新和维护
  • 了解记忆对 Agent 性能的影响

记忆的重要性

想象一个没有记忆的人：每次醒来都不知道自己是谁，不记得昨天发生了什么，不能从过去的错误中学习。这就是没有记忆的 Agent。有了记忆，Agent 可以：

  • 学习历史：记住过去的对话和决策
  • 改进决策：基于历史经验做出更好的选择
  • 个性化服务：记住用户的偏好和需求
  • 上下文理解：理解当前对话与历史的关系

有记忆 vs 无记忆

方面 无记忆 Agent 有记忆 Agent
用户体验 每次都要重复说明 记住用户信息，提供个性化服务
学习能力 无法学习和改进 从交互中学习，不断改进
上下文理解 只能理解当前对话 理解完整的对话历史
成本 每次都要重新推理 可以重用之前的结果

记忆存储的实现

class MemoryStore: """记忆存储接口""" def save(self, memory: Memory) -> str: """保存记忆，返回记忆 ID""" pass def load(self, memory_id: str) -> Memory: """加载指定 ID 的记忆""" pass # 具体实现：内存存储 class InMemoryStore(MemoryStore): def __init__(self): self.memories = {} def save(self, memory: Memory) -> str: memory_id = str(uuid.uuid4()) self.memories[memory_id] = memory return memory_id # 具体实现：数据库存储 class DatabaseStore(MemoryStore): def __init__(self, db_connection): self.db = db_connection def save(self, memory: Memory) -> str: self.db.insert("memories", memory.to_dict()) return memory.id
关键点： 选择合适的存储后端很重要。内存存储快但容量有限，数据库存储容量大但速度较慢。可以使用混合方案：热数据在内存，冷数据在数据库。

记忆的检索

存储记忆只是第一步，关键是如何高效地检索相关的记忆。检索算法的好坏直接影响 Agent 的决策质量。

检索方法

检索方法 说明 适用场景
精确匹配 根据 ID 或关键字精确查找 查找特定的记忆
相似度检索 根据语义相似度查找相关记忆 找到相关的历史对话
时间范围 根据时间范围检索记忆 查找最近的对话
混合检索 结合多种检索方法 复杂的查询需求

向量检索的实现

class VectorMemoryRetriever: def __init__(self, embedding_model, vector_db): self.embedding = embedding_model self.db = vector_db def retrieve(self, query: str, top_k: 5) -> List[Memory]: """根据查询检索相关的记忆""" # 1. 将查询转换为向量 query_vector = self.embedding.encode(query) # 2. 在向量数据库中搜索相似向量 results = self.db.search(query_vector, top_k=top_k) # 3. 返回相关的记忆 return [Memory.from_dict(r) for r in results]
示例： 记忆检索的实际应用
# 用户问："我上次说的那个项目怎么样了？" query = "项目进展" # Agent 检索相关的记忆 related_memories = retriever.retrieve(query, top_k=3) # 返回最相关的 3 条记忆 for memory in related_memories: print(memory.content)

记忆更新的实现

class MemoryManager: def update_memory(self, memory_id: str, updates: Dict): """更新记忆""" memory = self.store.load(memory_id) memory.update(updates) self.store.save(memory) def consolidate_memories(self, memories: List[Memory]) -> Memory: """整合多条记忆为一条""" # 将相似的记忆合并，减少存储空间 consolidated = Memory( content=" ".join([m.content for m in memories]), timestamp=max([m.timestamp for m in memories]) ) return consolidated def cleanup_old_memories(self, days: 30): """清理旧记忆""" cutoff_date = datetime.now() - timedelta(days=days) self.store.delete_before(cutoff_date)
警告： 不要随意删除记忆。应该先进行备份或归档，然后再删除。记忆可能包含重要的历史信息。

记忆的类型

不同类型的记忆有不同的特点和用途。理解这些类型有助于设计更有效的记忆系统。

记忆类型 特点 存储时间 用途

记忆类型的实现

class Memory: """基础记忆类""" def __init__(self, content: str, memory_type: str): self.content = content self.memory_type = memory_type self.timestamp = datetime.now() self.access_count = 0 class FactMemory(Memory): """事实记忆""" def __init__(self, content: str, source: str): super().__init__(content, "fact") self.source = source class EventMemory(Memory): """事件记忆""" def __init__(self, content: str, participants: List[str]): super().__init__(content, "event") self.participants = participants class SemanticMemory(Memory): """语义记忆""" def __init__(self, content: str, entities: Dict, relations: List): super().__init__(content, "semantic") self.entities = entities self.relations = relations

性能优化

随着记忆数量的增加，检索和更新的性能会下降。需要采用多种优化技术来保持系统的高效。

缓存策略

使用缓存可以显著提高频繁访问的记忆的检索速度：

class CachedMemoryStore: """带缓存的记忆存储""" def __init__(self, backend_store, cache_size: 1000): self.backend = backend_store self.cache = OrderedDict() self.cache_size = cache_size def get(self, memory_id: str) -> Memory: # 先查缓存 if memory_id in self.cache: self.cache.move_to_end(memory_id) return self.cache[memory_id] # 从后端存储加载 memory = self.backend.load(memory_id) # 加入缓存 self.cache[memory_id] = memory if len(self.cache) > self.cache_size: self.cache.popitem(last=False) return memory

索引优化

为常用的查询字段建立索引，加快查询速度：

class IndexedMemoryStore: """带索引的记忆存储""" def __init__(self): self.memories = {} self.type_index = {} # 按类型索引 self.time_index = {} # 按时间索引 self.tag_index = {} # 按标签索引 def query_by_type(self, memory_type: str) -> List[Memory]: """快速查询指定类型的记忆""" return [self.memories[mid] for mid in self.type_index.get(memory_type, [])] def query_by_time_range(self, start: datetime, end: datetime) -> List[Memory]: """快速查询时间范围内的记忆""" results = [] for mid, memory in self.memories.items(): if start <= memory.timestamp <= end: results.append(memory) return results
优化建议： 根据实际使用情况选择合适的优化策略。热数据使用缓存，常用查询建立索引，定期清理过期数据。

最佳实践

1. 记忆大小控制

记忆太多会导致检索变慢，成本增加。应该定期清理过期的记忆，整合相似的记忆。

2. 隐私保护

记忆可能包含敏感信息。应该对敏感数据进行加密存储，限制访问权限。

3. 检索优化

使用合适的索引和缓存，提高检索速度。对于频繁访问的记忆，可以使用缓存。

4. 记忆质量

定期审查记忆的质量，删除错误或过时的信息。记忆的质量直接影响 Agent 的决策质量。

总结

长期记忆是 Agent 的知识库和学习能力的基础。通过合理的存储、检索和维护，Agent 可以不断积累经验、改进决策。本章深入讲解了记忆的重要性、存储方式、检索机制、更新策略、记忆类型和性能优化。

核心要点总结：
  • 记忆的重要性： 使 Agent 能够学习、改进、提供个性化服务
  • 存储层次： 短期记忆 → 工作记忆 → 长期记忆，选择合适的存储后端
  • 检索机制： 精确匹配、相似度检索、时间范围等多种方法
  • 记忆类型： 事实记忆、事件记忆、语义记忆、程序记忆各有用途
  • 更新维护： 定期更新、整合、清理记忆，保持系统健康
  • 性能优化： 缓存策略、索引优化、定期清理
  • 最佳实践： 控制大小、保护隐私、优化检索、保证质量
记忆系统建议： 记忆系统的设计应该平衡存储成本、检索速度和准确性。根据实际应用场景选择合适的存储后端和检索算法。同时要定期维护和优化记忆系统，确保其持续高效运行。
