🧠 五个你必须先记住的 DeerFlow 关键词

名词一句话理解类比
GatewayDeerFlow 默认运行时承载体，不只是外层 API系统总入口和总控台
Thread / Run对话状态和一次执行的统一语义单元工单和工单的一次处理过程
Skill能力包与工作说明，不等于 tool 本身团队内部 SOP 和手册
Sandbox围绕虚拟路径契约组织的受控文件/命令边界受控工作区，不是随意碰主机
Sub-Agent通过 task 工具触发的结构化委派单元明确派给另一个小组的子任务
💡 怎么用这套文档效果最好

1. 每读完一章，至少打开该章点名的 2-3 个核心文件，不要只停留在 HTML 页面。

2. 对所有“边界”保持敏感：哪些是当前已实现，哪些只是兼容层，哪些是 RFC 设计方向。

3. 如果你是来做二次开发或排障，优先读第 03、10、15、20、40、41 章，再按需回看单项能力章节。
