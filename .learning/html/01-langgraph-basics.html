1. 什么是LangGraph？

LangGraph是由LangChain团队开发的一个框架， 用于构建具有明确控制流的Agent应用。 它提供了一种声明式的方式来定义Agent的执行逻辑。

LangGraph的核心目标：
提供一个清晰、可控、可扩展的方式来构建复杂的Agent系统， 特别是那些需要多步骤推理和决策的系统。

LangGraph的主要特点

  • 显式控制流： 你可以清晰地定义Agent在不同状态下的行为
  • 状态管理： 系统自动管理Agent的状态变化
  • 可视化： 可以生成Graph的可视化表示，便于理解和调试
  • 持久化： 支持检查点（Checkpoint），可以保存和恢复执行状态
  • 可扩展性： 易于添加新的节点和边，扩展功能
  • LangSmith集成： 与LangSmith无缝集成，便于监控和调试

LangGraph解决的问题

问题1：控制流不清晰
在没有LangGraph的情况下，Agent的执行逻辑散落在各个函数中， 很难理解整个流程。
问题2：状态管理复杂
需要手动管理Agent的各种状态，容易出错。
问题3：难以调试
当Agent出现问题时，很难追踪执行过程。
问题4：难以扩展
添加新的功能需要修改核心代码。

2. 为什么DeerFlow选择LangGraph？

DeerFlow 2.0是一个完全的重写，选择LangGraph作为核心框架有几个重要原因。

1. 成熟的生态

LangGraph由LangChain团队维护，有完整的文档、活跃的社区和持续的更新。 这确保了框架的稳定性和可靠性。

2. 强大的抽象能力

LangGraph提供了Graph、Node、Edge等抽象， 这些抽象非常适合表示Agent的执行流程。

3. 天然支持多Agent

LangGraph的设计天然支持多Agent系统。 DeerFlow中的Sub-Agents就是基于LangGraph的Graph嵌套实现的。

4. 优秀的可观测性

LangGraph与LangSmith和Langfuse的集成， 使得Agent的执行过程可以被完整地追踪和监控。

5. 灵活的状态管理

LangGraph的状态管理机制非常灵活， 可以轻松处理复杂的状态转移。

总结： LangGraph不仅提供了技术上的优势， 更重要的是它的设计哲学与DeerFlow的目标完全一致： 构建清晰、可控、可扩展的Agent系统。

3. 核心概念：Graph、Node、Edge、State

理解这四个核心概念是掌握LangGraph的基础。

概念1：Graph（图）

Graph是LangGraph的最基本单位， 它代表了Agent的整个执行流程。

Graph = 节点 + 边 + 状态
一个Graph由多个Node组成，Node之间通过Edge连接， 整个Graph共享一个State。

Graph的特点：

  • 有明确的入口点（START）和出口点（END）
  • 可以有条件分支（conditional edges）
  • 可以有循环（cycles）
  • 可以嵌套（一个Graph可以包含另一个Graph）

概念2：Node（节点）

Node代表执行流程中的一个步骤。 每个Node是一个可调用的函数或对象。

Node的例子：
- 调用LLM的Node
- 执行工具的Node
- 处理结果的Node
- 做出决策的Node

Node的特点：

  • 接收当前的State作为输入
  • 执行某些操作
  • 返回更新后的State
  • 可以是同步的或异步的

概念3：Edge（边）

Edge定义了Node之间的连接关系。 它指定了从一个Node到另一个Node的转移。

Edge的类型：

类型 说明 例子
普通Edge 无条件转移，总是执行 从Node A到Node B
条件Edge 根据条件决定转移 如果结果为"continue"，转到Node B；否则转到Node C
循环Edge 可以返回到之前的Node 如果需要重试，返回到"调用LLM"的Node

概念4：State（状态）

State是Graph中所有Node共享的数据结构。 它记录了Agent执行过程中的所有信息。

State的作用：
1. 存储Agent的输入和输出
2. 记录中间结果
3. 追踪Agent的执行历史
4. 在Node之间传递数据

State的特点：

  • 通常是一个字典或TypedDict
  • 所有Node都可以读写State
  • State的变化被自动追踪
  • 可以被持久化（保存和恢复）
State的例子：
{ "messages": [...], # 对话历史 "current_step": "thinking", # 当前步骤 "tools_used": [...], # 已使用的工具 "intermediate_results": {...}, # 中间结果 "final_output": None # 最终输出 }

4. 工作流程：从定义到执行

让我们看看一个LangGraph应用的完整生命周期。

步骤1：定义State

首先，定义Graph将使用的State结构。

from typing import TypedDict, List class AgentState(TypedDict): messages: List[str] current_step: str tools_used: List[str] final_output: str

步骤2：定义Node

然后，定义Graph中的各个Node。

def call_llm(state: AgentState) -> AgentState: """调用LLM的Node""" # 处理当前消息 response = llm.invoke(state["messages"]) # 更新State state["messages"].append(response) state["current_step"] = "tool_calling" return state def execute_tool(state: AgentState) -> AgentState: """执行工具的Node""" # 从最后一条消息中提取工具调用 tool_call = extract_tool_call(state["messages"][-1]) # 执行工具 result = execute(tool_call) # 更新State state["messages"].append(result) state["tools_used"].append(tool_call["tool_name"]) return state

步骤3：定义Edge

定义Node之间的连接关系。

def should_continue(state: AgentState) -> str: """决定是否继续执行""" last_message = state["messages"][-1] if "tool_call" in last_message: return "execute_tool" else: return "end" # 添加Edge graph.add_edge("start", "call_llm") graph.add_edge("call_llm", "route") graph.add_conditional_edges( "route", should_continue, { "execute_tool": "execute_tool", "end": "end" } ) graph.add_edge("execute_tool", "call_llm")

步骤4：编译Graph

编译Graph以生成可执行的应用。

app = graph.compile()

步骤5：执行Graph

执行Graph来处理输入。

initial_state = { "messages": ["用户问题"], "current_step": "start", "tools_used": [], "final_output": None } result = app.invoke(initial_state) print(result["final_output"])

5. LangGraph vs LangChain

LangGraph和LangChain都是LangChain生态的一部分， 但它们的用途不同。

方面 LangChain LangGraph
主要用途 构建LLM应用的通用框架 构建具有明确控制流的Agent
抽象层级 高层抽象（Chain、Runnable等） 低层抽象（Graph、Node、Edge）
控制流 隐式的，通过Chain的组合 显式的，通过Graph定义
适用场景 简单的顺序流程 复杂的多步骤推理
可视化 有限的可视化支持 完整的Graph可视化
状态管理 隐式的 显式的

何时使用LangChain，何时使用LangGraph

使用LangChain的场景：
✓ 简单的顺序流程
✓ 快速原型开发
✓ 不需要复杂的控制流
✓ 需要使用LangChain的其他组件（如RAG）
使用LangGraph的场景：
✓ 复杂的多步骤推理
✓ 需要明确的控制流
✓ 需要循环和条件分支
✓ 需要完整的可观测性
✓ 构建生产级应用
重要： LangGraph和LangChain不是竞争关系， 而是互补关系。LangGraph通常会使用LangChain的组件（如Runnable）。

6. 简单示例：Hello World Graph

让我们看一个最简单的LangGraph应用。

from langgraph.graph import StateGraph, START, END from typing import TypedDict # 定义State class SimpleState(TypedDict): message: str # 定义Node def node_a(state: SimpleState) -> SimpleState: print(f"Node A: {state['message']}") state['message'] += " -> A" return state def node_b(state: SimpleState) -> SimpleState: print(f"Node B: {state['message']}") state['message'] += " -> B" return state # 构建Graph graph = StateGraph(SimpleState) graph.add_node("node_a", node_a) graph.add_node("node_b", node_b) # 添加Edge graph.add_edge(START, "node_a") graph.add_edge("node_a", "node_b") graph.add_edge("node_b", END) # 编译 app = graph.compile() # 执行 result = app.invoke({"message": "Hello"}) print(result['message']) # 输出: Hello -> A -> B

代码解析

1. 定义State：

SimpleState是一个TypedDict，包含一个message字段。 这就是Graph中所有Node共享的数据。

2. 定义Node：

node_a和node_b都是简单的函数， 接收State作为输入，返回更新后的State。

3. 构建Graph：

使用StateGraph创建一个Graph， 然后添加Node和Edge。

4. 执行：

调用app.invoke()来执行Graph。

7. 复杂示例：Agent Graph

现在让我们看一个更复杂的例子， 展示一个简单的Agent如何使用LangGraph实现。

from langgraph.graph import StateGraph, START, END from typing import TypedDict, List from langchain_openai import ChatOpenAI class AgentState(TypedDict): messages: List[dict] current_step: str final_answer: str def call_llm(state: AgentState) -> AgentState: """调用LLM""" llm = ChatOpenAI(model="gpt-4") response = llm.invoke(state["messages"]) state["messages"].append({ "role": "assistant", "content": response.content }) state["current_step"] = "check_response" return state def check_response(state: AgentState) -> AgentState: """检查LLM的响应""" last_message = state["messages"][-1]["content"] if "I need to search" in last_message: state["current_step"] = "search" else: state["current_step"] = "end" return state def search(state: AgentState) -> AgentState: """执行搜索""" # 这里应该调用搜索工具 state["messages"].append({ "role": "system", "content": "搜索结果..." }) state["current_step"] = "call_llm" return state def finalize(state: AgentState) -> AgentState: """最终化结果""" state["final_answer"] = state["messages"][-1]["content"] return state # 构建Graph graph = StateGraph(AgentState) graph.add_node("call_llm", call_llm) graph.add_node("check_response", check_response) graph.add_node("search", search) graph.add_node("finalize", finalize) # 添加Edge graph.add_edge(START, "call_llm") graph.add_edge("call_llm", "check_response") # 条件Edge def route(state: AgentState) -> str: if state["current_step"] == "search": return "search" else: return "finalize" graph.add_conditional_edges("check_response", route) graph.add_edge("search", "call_llm") graph.add_edge("finalize", END) # 编译和执行 app = graph.compile() initial_state = { "messages": [{"role": "user", "content": "什么是Agent？"}], "current_step": "start", "final_answer": "" } result = app.invoke(initial_state) print(result["final_answer"])

这个例子展示了什么

  • 多个Node： call_llm、check_response、search、finalize
  • 条件分支： 根据LLM的响应决定是否执行搜索
  • 循环： 搜索后可以回到call_llm
  • 状态管理： State在各个Node之间传递和更新

1. LangGraph是什么： 一个用于构建具有明确控制流的Agent框架
2. 核心概念： Graph、Node、Edge、State
3. Graph = 节点 + 边 + 状态
