1. DeerFlow是什么？

DeerFlow（鹿流）是由字节跳动开源的一个生产级Agent框架。 它的全称是"Deep Exploration and Efficient Research Flow"， 但在2.0版本中，它已经演变成了一个通用的Super Agent Harness。

DeerFlow 2.0的定位：
一个开源的超级Agent框架，能够编排Sub-Agents、Memory和Sandboxes， 通过可扩展的Skills来完成几乎任何事情。

DeerFlow的主要特点

  • 开箱即用： 内置了大量功能，无需从零开始
  • 完全可扩展： 可以轻松添加自定义Skills、Tools和Agents
  • 生产就绪： 经过字节跳动等大厂的实际使用验证
  • 多渠道支持： 支持Web UI、API、IM（Telegram、Slack等）
  • 完整的可观测性： 与LangSmith、Langfuse集成
  • 安全隔离： 支持多种Sandbox模式
  • 长期记忆： 内置记忆系统，支持跨会话学习

DeerFlow的版本演进

版本 定位 主要特点 状态
1.x Deep Research框架 专注于研究任务 维护中
2.0 Super Agent Harness 通用Agent框架，完全重写 活跃开发
重要： 本教程主要讲解DeerFlow 2.0。 如果你使用的是1.x版本，请参考官方文档。

2. 设计哲学：从Deep Research到Super Agent Harness

理解DeerFlow的设计哲学，有助于我们更好地使用和扩展它。

DeerFlow 1.x：Deep Research框架

DeerFlow最初是为了解决一个特定问题： 如何让Agent能够进行深度研究，生成高质量的研究报告。

1.x的典型工作流：
1. 接收研究课题
2. 制定研究计划
3. 执行多个搜索任务
4. 整理和分析信息
5. 生成研究报告

DeerFlow 2.0：Super Agent Harness

在2.0版本中，DeerFlow的目标发生了变化。 团队意识到，DeerFlow不仅仅是一个研究工具， 而是一个通用的Agent框架。

2.0的核心理念：
"一个框架，无限可能"

DeerFlow提供了构建任何类型Agent所需的基础设施， 用户可以通过Skills和Tools来定制功能。

从1.x到2.0的变化

方面 1.x 2.0
定位 Deep Research框架 通用Agent框架
架构 基于Chain的顺序流程 基于LangGraph的Graph流程
可扩展性 有限，需要修改核心代码 高，通过Skills和Tools扩展
Sub-Agents 不支持 原生支持
Memory 简单的会话记忆 完整的长期记忆系统
Sandbox 基础支持 多种模式，完整隔离

设计原则

1. 电池包含（Batteries Included）
DeerFlow开箱即用，内置了大量功能， 用户可以立即开始构建Agent。
2. 完全可扩展（Fully Extensible）
虽然内置功能完整，但用户可以轻松替换或扩展任何部分。
3. 生产就绪（Production Ready）
DeerFlow不是一个实验性框架，而是经过大规模验证的生产级系统。
4. 清晰的抽象（Clear Abstractions）
DeerFlow定义了清晰的概念（Skills、Tools、Agents等）， 使得系统易于理解和使用。

3. 核心组件

DeerFlow由几个核心组件组成，每个组件都有明确的职责。

🧠 Agent Runtime

Agent的执行引擎，负责Agent的生命周期管理和执行。

职责： 状态管理、工具调用、决策制定

🛠️ Skills

Agent的能力模块，定义了Agent能做什么。

职责： 提供高级能力，如研究、代码生成等

🔧 Tools

Agent的工具集，提供具体的操作能力。

职责： 网络搜索、文件操作、代码执行等

📦 Sandbox

隔离执行环境，保证安全性。

职责： 代码执行隔离、文件系统隔离

💾 Memory

长期记忆系统，支持跨会话学习。

职责： 存储、检索、更新用户信息

🌐 Gateway

API网关，提供HTTP接口。

职责： 请求路由、认证、响应格式化

组件之间的关系

Agent Runtime（核心）

Agent Runtime是DeerFlow的核心， 它协调其他所有组件的工作。

  • 使用Skills定义Agent的能力
  • 使用Tools执行具体操作
  • 使用Sandbox隔离代码执行
  • 使用Memory存储长期信息
  • 通过Gateway暴露API

详细的步骤

步骤1：用户输入
用户通过Web UI、API或IM发送请求。 例如："请帮我分析这份数据"
步骤2：Gateway处理
Gateway接收请求，进行认证和授权， 然后将请求转发给Agent Runtime。
步骤3：Agent Runtime分析
Agent Runtime（基于LangGraph）分析请求， 决定需要使用哪些Skills或Tools。
步骤4：Skills/Tools执行
Agent调用相应的Skills（如数据分析Skill）或Tools（如代码执行）。
步骤5：Sandbox隔离
如果需要执行代码，Sandbox提供隔离的执行环境。
步骤6：Memory更新
Agent的执行结果和用户信息被保存到Memory中， 用于未来的请求。
步骤7：输出
Agent生成最终答案，通过Gateway返回给用户。

一个具体的例子

例子：用户请求"写一个Python函数"

  1. 输入： "请写一个Python函数来计算斐波那契数列"
  2. Gateway： 验证用户身份，创建新的会话
  3. Agent Runtime： 分析请求，决定使用CodeGen Skill
  4. CodeGen Skill： 调用LLM生成代码
  5. Sandbox： 执行生成的代码进行测试
  6. Agent Runtime： 收集结果，生成最终答案
  7. Memory： 保存用户的编程偏好
  8. 输出： 返回代码和执行结果

6. 与其他框架的对比

理解DeerFlow与其他框架的区别，有助于你选择合适的工具。

框架 定位 优点 缺点
DeerFlow 通用Agent框架 功能完整、生产就绪、可扩展 学习曲线陡峭
LangChain LLM应用框架 灵活、生态丰富 不专注于Agent
AutoGPT 自主Agent 自主性强 不稳定、难以控制
CrewAI 多Agent框架 多Agent协作 功能有限
MetaGPT 软件开发Agent 专注于代码生成 应用场景有限

DeerFlow的竞争优势

1. 完整性： 不仅是框架，还包括完整的生产系统
2. 可靠性： 经过字节跳动等大厂的实际使用验证
3. 可扩展性： 通过Skills和Tools轻松扩展
4. 多渠道： 支持Web、API、IM等多种接入方式
5. 安全性： 完整的Sandbox隔离机制

7. 适用场景

DeerFlow适合哪些场景？让我们看一些例子。

适合的场景

📊 数据分析Agent
接收数据分析需求 → 查询数据库 → 执行分析 → 生成报告
🔬 研究Agent
接收研究课题 → 网络搜索 → 信息整理 → 生成报告
💻 编程Agent
接收功能需求 → 代码生成 → 测试执行 → Bug修复
📝 内容创作Agent
接收创作需求 → 信息收集 → 内容创作 → 质量审查
🤖 客服Agent
接收用户问题 → 知识库查询 → 问题分类 → 生成回复
🎯 任务自动化Agent
接收任务 → 任务分解 → 执行子任务 → 结果汇聚

不适合的场景

1. 简单的Chatbot
如果只需要简单的对话功能，使用LangChain或其他轻量级框架更合适。
2. 实时系统
DeerFlow的Agent执行可能比较慢，不适合对延迟要求极高的系统。
3. 成本敏感的应用
DeerFlow会调用LLM多次，成本可能较高。
4. 完全自主的系统
DeerFlow需要明确的目标和约束，不适合完全自主的系统。

📝 关键要点总结

1. DeerFlow是什么： 生产级的通用Agent框架
2. 核心组件： Agent Runtime、Skills、Tools、Sandbox、Memory、Gateway
3. 设计哲学： 电池包含 + 完全可扩展 + 生产就绪
4. 架构分层： 表现层 → 网关层 → 执行层 → 能力层 → 隔离层 → 存储层
5. 数据流： 用户输入 → Gateway → Agent Runtime → Skills/Tools → Sandbox → Memory → 输出
6. 竞争优势： 完整、可靠、可扩展、多渠道、安全
